# UTRRWManager 実機確認メモ

## 目的

このファイルは、UTRRWManager を実機確認、送受信ログ取得、Pythonサンプルとの比較に使うためのメモです。

## 参照元

- `docs/manuals/original/TDR-MNL-UTRRWManagerV151-101.pdf`

## 位置づけ

- UTRRWManager は、実機確認用および正解ログ取得用の公式ユーティリティとして扱う。
- 通信コマンド仕様の正は、引き続き `docs/protocol/` とする。
- UTRRWManager の `[cmt]` は解析補助情報として扱う。
- Pythonサンプルとの比較では、対象コマンドごとに `[cmd]` / `[TX]` / `[RX]` / `[cmt]` を切り出して比較する。

> 根拠: UTRRWManager取扱説明書 第1章 1.5「対応リーダライタ」本文p.1
> 根拠: UTRRWManager取扱説明書 第3章 3.1.1「コマンドモード」本文p.13
> 根拠: UTRRWManager取扱説明書 第3章 3.10.2「環境設定」本文p.192

## UTRRWManagerの役割

- UTRシリーズのリーダライタと接続し、リーダライタ制御コマンド、設定コマンド、RFタグ通信コマンドの実行結果を確認する。
- TCP/IP接続成功時、UTRRWManager側で動作モード読み取り、ROMバージョン読み取り等が行われる場合がある。
- 送受信ログに、実行コマンド、TX、RX、解析コメントを時系列で表示できる。
- 起動時や終了時に設定ファイルを読み書きするため、管理者として実行する必要がある。

> 根拠: UTRRWManager取扱説明書 第2章 2.2「UTRRWManager の起動方法」本文p.5
> 根拠: UTRRWManager取扱説明書 第2章 2.4「TCP/IP で接続する」本文p.11
> 根拠: UTRRWManager取扱説明書 第3章 3.1.1「コマンドモード」本文p.13

## TCP/IP接続手順

1. UTRRWManagerを起動する。
2. ダイアログが「シリアルポートのオープン」の場合は、`LANタイプに切り替え` を押す。
3. リーダライタのIPアドレスとポート番号を入力する。
4. `接続` を押す。
5. 通信確立に成功した場合、送受信ログに動作モード読み取り、ROMバージョン読み取り等が表示される。
6. 通信確立できない場合は `TCP/IP 接続中` が表示され、約10秒後に `NG： Port Open` が表示される。

> 根拠: UTRRWManager取扱説明書 第2章 2.4「TCP/IP で接続する」本文p.8-p.11

## 出荷時IPアドレスとポート

| 項目 | 値 |
|---|---|
| IPアドレス | `192.168.0.1` |
| TCPポート番号 | `9004` |
| TCP/IP動作モード(PC側) | クライアント |

実機の現在設定が出荷時設定と一致するとは限らないため、実機テスト前に現在値を確認する。

> 根拠: UTRRWManager取扱説明書 第2章 2.4「TCP/IP で接続する」本文p.8

## 送受信ログの見方

| 表示 | 意味 |
|---|---|
| `[cmd]` | 実行したコマンド名 |
| `[TX]` | 上位機器からリーダライタへの送信コマンド |
| `[RX]` | リーダライタから上位機器へのレスポンス |
| `[cmt]` | レスポンスの解析結果コメント |

- `[TX]` と `[RX]` はPythonサンプルの送信HEX、受信HEXとの比較対象とする。
- `[cmt]` は解析補助情報として扱い、通信仕様の根拠にはしない。
- 送受信ログは最大2000件まで表示され、2000件を超えると内容がクリアされる。
- 送受信ログの内容はテキストデータとしてコピーできる。

> 根拠: UTRRWManager取扱説明書 第3章 3.1.1「コマンドモード」本文p.13

## ROMバージョン読み取りの確認方法

- TCP/IP接続成功時に、UTRRWManager側でROMバージョン読み取り等が行われる場合がある。
- 送受信ログから、ROMバージョン読み取りの `[cmd]` / `[TX]` / `[RX]` / `[cmt]` を切り出す。
- PythonサンプルのROMバージョン確認コマンドの送受信HEXと比較する。
- ROMバージョン読み取りの通信仕様は `docs/protocol/command_list.md` を正とする。

> 根拠: UTRRWManager取扱説明書 第2章 2.4「TCP/IP で接続する」本文p.11
> 根拠: UTRRWManager取扱説明書 第3章 3.1.1「コマンドモード」本文p.13

## Inventory 1回実行時の確認方法

- メイン画面の `[●Inventory]` ボタンを押すと、`UHF_Inventory` コマンドが1回実行される。
- コマンドモード時に、メニューから `[RF タグ通信コマンド]` の `[UHF_Inventory]` を押した時と同じ動作を行う。
- 読み取り結果は `[受信データ一覧]` と `[送受信ログ]` に表示される。
- Pythonサンプルとの比較では、Inventory 1回分の `[cmd]` / `[TX]` / `[RX]` / `[cmt]` を切り出す。

> 根拠: UTRRWManager取扱説明書 第3章 3.1.7「Inventory コマンドボタン」本文p.22
> 根拠: UTRRWManager取扱説明書 第3章 3.4.1「UHF_Inventory」本文p.107

## ログ保存方法

### 送受信ログの手動保存

- `[ファイル]` メニューから `[送受信ログを保存]` を実行すると、送受信ログの内容をテキストデータ `*.txt` として保存できる。
- ショートカットキーは `Shift + Ctrl + s`。
- デフォルトのファイル名には現在時刻が入る。

> 根拠: UTRRWManager取扱説明書 第3章 3.8.2「送受信ログを保存」本文p.185

### 送受信ログの自動保存

- `[ヘルプ]` メニューの `[環境設定]` で、送受信ログを自動的にファイルへ保存する設定ができる。
- 自動保存を有効にした場合、起動時に指定フォルダへ `YYYYMMDDhhmmss.txt` 形式のテキストファイルが生成される。

> 根拠: UTRRWManager取扱説明書 第3章 3.10.2「環境設定」本文p.198

## Pythonサンプルとの比較に使うTX/RXログ

比較対象コマンドごとに、以下を1セットとして保存する。

- `[cmd]`: コマンド名
- `[TX]`: UTRRWManagerが送信したHEX
- `[RX]`: リーダライタが返したHEX
- `[cmt]`: UTRRWManagerの解析補助コメント

比較対象の初期候補:

- ROMバージョン読み取り
- RAM指定のコマンドモード設定
- UHF_Inventory 1回

`[TX]` / `[RX]` は通信データ比較に使い、`[cmt]` は人間が確認する補助情報として扱う。

> 根拠: UTRRWManager取扱説明書 第3章 3.1.1「コマンドモード」本文p.13
> 根拠: UTRRWManager取扱説明書 第3章 3.10.2「環境設定」本文p.192

## 初期確認では使わない操作

- FLASH設定をファイルに保存
- 送受信ログの自動保存

これらは読み出しやログ取得に近い操作だが、初期確認では手順を単純にするため使わない。

> 根拠: UTRRWManager取扱説明書 第3章 3.8.3「リーダライタの FLASH 設定をファイルに保存」本文p.186
> 根拠: UTRRWManager取扱説明書 第3章 3.10.2「環境設定」本文p.198
