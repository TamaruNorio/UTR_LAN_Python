## ROMバージョン確認コマンド

送信例:

```text
02 00 4F 01 90 03 E5 0D
```

正常応答の要点:

- ACK `30h`
- データ長 `0Ah`
- データ部は `90h` + メジャーバージョン1バイト + マイナーバージョン3バイト + シリーズ名5バイト
- バージョン番号とシリーズ名はASCIIコードとして返る

> 根拠: 第7章 7.3.8「ROM バージョンの読み取り」本文p.145

## コマンドモード設定コマンド

- コマンド: `4Eh`
- データ長: `07h`
- 書き込み対象:
  - `00h`: RAMへの書き込み
  - `10h`: FLASHへの書き込み
- リーダライタ動作モード:
  - `00h`: コマンドモード
  - `65h`: UHF連続インベントリモード
  - `66h`: UHF連続インベントリリードモード

初期実機テストでは `00h: RAMへの書き込み` のみ使用し、`10h: FLASHへの書き込み` は禁止する。

> 根拠: 第7章 7.4.16「リーダライタ動作モードの書き込み」本文p.188-p.189

## Inventory系コマンド

### UHF_Inventory

- コマンド: `55h`
- データ長: `01h`
- データ部: `10h`

送信例:

```text
02 00 55 01 10 03 6B 0D
```

RFタグを読み取った場合、RFタグデータレスポンスがタグ枚数分返り、その後、読み取り完了レスポンスが返る。RFタグが読めなかった場合は、読み取り完了レスポンスのみ返る。

> 根拠: 第7章 7.5.1「UHF_Inventory」本文p.262-p.265

### UHF_InventoryRead

- コマンド: `55h`
- データ長: `07h`
- データ部先頭: `14h`
- 指定MemBank、TID付加、読み取り開始Wordアドレス、読み取りWord数を指定する。

> 根拠: 第7章 7.5.2「UHF_InventoryRead」本文p.266-p.270

## ブザー制御コマンド

- コマンド: `42h`
- データ長: `02h`
- データ部:
  - 応答要求 `00h`: 応答を要求しない
  - 応答要求 `01h`: 応答を要求する
  - ブザー音 `00h`から`08h`

応答要求なしの場合でも、SUM値エラーなどではNACK応答が返る。

初期実機テストでは、ブザー制御はユーザーが明示的に許可するまで実行しない。

> 根拠: 第7章 7.3.2「ブザーの制御」本文p.124-p.125

## 正常応答フォーマット

ACK応答はコマンド部 `30h` で判定する。データ部の内容はコマンドごとに異なる。

例: ROMバージョン読み取りでは `90h` に続いてASCIIのバージョン・シリーズ名が返る。

> 根拠: 第6章 6.1.4「コマンド逆引き表」本文p.106
> 根拠: 第7章 7.3.8「ROM バージョンの読み取り」本文p.145

## 異常応答フォーマット

NACK応答はコマンド部 `31h`、データ長 `0Ah` で返る。

| エラーコード1 | シンボル | 意味 |
|---|---|---|
| `01h` | CMD_CRC_ERROR | RFタグ受信データのCRC不一致。 |
| `02h` | CMD_TIME_OVER | RFタグ受信データが途中で途切れた。 |
| `03h` | CMD_RX_ERROR | アンチコリジョン処理中のエラー。 |
| `04h` | CMD_RXBUSY_ERROR | RFタグから応答なし。 |
| `07h` | CMD_ERROR | コマンド実行中の内部エラー。 |
| `0Ah` | CMD_UHF_IC_ERROR | 内蔵チップが返すRFタグアクセス系エラー。 |
| `42h` | SUM_ERROR | 上位機器から送信されたSUM値が不正。 |
| `44h` | FORMAT_ERROR | フォーマットまたはパラメータ不正。 |
| `60h` | CMD_LBT_ERROR | キャリアセンス時のタイムアウト。 |
| `64h` | HARDWARE_ERROR | ハードウェア内部異常。 |
| `68h` | CMD_ANT_ERROR | アンテナ断線検知時のRFタグ通信エラー。 |

> 根拠: 第7章 7.6「NACK レスポンスとエラーコード」本文p.316-p.318

## タイムアウト時の扱い

- RFタグ通信では、RFタグから応答が返らない場合や受信できない場合、リーダライタ側が一定時間でタイムアウトし、リトライやNACKレスポンス送信などの次処理へ移る。
- コマンドタイムアウト時間の初期値は 20ms。
- 設定可能範囲は 0msから20ms。
