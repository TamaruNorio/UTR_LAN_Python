# フレーム構造

## 概要

UTR-S201シリーズのコマンドおよびレスポンスは、共通のバイナリフレーム形式で送受信する。

> 根拠: 第5章 5.1「コマンド／レスポンスの通信フォーマット」本文p.98
> 根拠: 第5章 5.2「通信フォーマットの詳細」本文p.99

## 基本フレーム形式

| 項目 | バイト数 | 内容 |
|---|---:|---|
| STX | 1 | `02h`。パケット先頭。 |
| アドレス | 1 | 通常送信時は `00h`。 |
| コマンド | 1 | コマンドコード。 |
| データ長 | 1 | データ部のバイト数。`00h`から`FFh`。 |
| データ部 | 0から255 | コマンドごとに異なる。 |
| ETX | 1 | `03h`。パケット終端。 |
| SUM | 1 | STXからETXまでのSUM値。 |
| CR | 1 | `0Dh`。改行コード。 |

> 根拠: 第5章 5.1「コマンド／レスポンスの通信フォーマット」本文p.98
> 根拠: 第5章 5.2「通信フォーマットの詳細」本文p.99

## フレーム長

パケット全体の長さは `データ長 + 7` バイト。

```text
全体長 = STX(1) + アドレス(1) + コマンド(1) + データ長(1)
       + データ部(N) + ETX(1) + SUM(1) + CR(1)
       = N + 7
```

> 根拠: 第5章 5.2「通信フォーマットの詳細」本文p.99

## STX / ETX / CR の扱い

- STX は `02h`。
- ETX は `03h`。
- CR は `0Dh`。
- SUM計算対象は STX から ETX まで。
- SUM自身とCRはSUM計算対象に含めない。

> 根拠: 第5章 5.2「通信フォーマットの詳細」本文p.99
> 根拠: 第5章 5.3「SUM の計算方法」本文p.100

## アドレス

コマンド送信時、通常は `00h` を設定する。

`00h` を指定した場合、リーダライタIDに関わらずコマンド処理が実行され、レスポンスが返る。`00h` 以外を指定した場合、指定IDと一致するリーダライタのみが処理する。

レスポンス受信時は原則としてリーダライタIDが入る。ただし、アンテナID出力が有効で、RFタグ読み取りデータを返すレスポンスの場合は、RFタグを読み取ったアンテナ番号がセットされる。

> 根拠: 第5章 5.2「通信フォーマットの詳細」本文p.99

## 送信フレーム

送信フレームは以下を基本とする。

```text
02 [アドレス] [コマンド] [データ長] [データ部...] 03 [SUM] 0D
```

通常のアドレスは `00h`。

> 根拠: 第5章 5.1「コマンド／レスポンスの通信フォーマット」本文p.98
> 根拠: 第5章 5.2「通信フォーマットの詳細」本文p.99

## 正常応答フォーマット

正常応答は、コマンド部が `30h` の ACK 応答として返る。

```text
02 [アドレス] 30 [データ長] [データ部...] 03 [SUM] 0D
```

データ部の内容は実行したコマンドごとに異なる。前段で実行したコマンドの詳細コマンドがデータ部に付加される場合がある。

> 根拠: 第6章 6.1.4「コマンド逆引き表」本文p.106

## RFタグ読み取りデータ応答

RFタグの読み取りデータは、ACK完了応答とは別に、コマンド部 `6Ch` のレスポンスとして返る。

UHF_Inventory のRFタグデータ応答では、データ部の先頭が `09h` となり、RSSI、ANGLE、PC+EPC が続く。

> 根拠: 第7章 7.5.1「UHF_Inventory」本文p.263-p.265

## 異常応答フォーマット

異常応答は、コマンド部が `31h` の NACK 応答として返る。

```text
02 [アドレス] 31 0A [詳細コマンド] [エラーコード1] [エラーコード2] [エラーコード3] [エラーコード4] [予約5バイト] 03 [SUM] 0D
```

NACKのデータ長は `0Ah`。

| データ部 | 内容 |
|---|---|
| 1バイト目 | 詳細コマンド |
| 2バイト目 | エラーコード1 |
| 3バイト目 | エラーコード2 |
| 4バイト目 | エラーコード3 |
| 5バイト目 | エラーコード4 |
| 6から10バイト目 | 予約領域。上位側では無視する。 |

> 根拠: 第7章 7.6「NACK レスポンスとエラーコード」本文p.316

## ACK / NACK 判定

| 判定 | コマンド部 |
|---|---|
| ACK | `30h` |
| NACK | `31h` |

> 根拠: 第6章 6.1.4「コマンド逆引き表」本文p.106
> 根拠: 第7章 7.6「NACK レスポンスとエラーコード」本文p.316

## 受信時の実装メモ

- 受信データはバイナリとして扱う。
- STX、ETX、CR、データ長、SUMを検証する。
- データ長から期待フレーム長を算出する。
- ACK / NACK / RFタグ読み取りデータをコマンド部で判定する。
- 自動読み取りモードでは、上位コマンドへのレスポンス前にRFタグデータが返る場合がある。

> 根拠: 第5章 5.4「コマンドレスポンス」本文p.101
