# SUM / チェックサム計算

## 概要

UTR-S201シリーズのフレームでは、STXからETXまでのデータを対象にSUM値を計算する。

> 根拠: 第5章 5.3「SUM の計算方法」本文p.100

## 計算対象範囲

SUM計算に含める:

- STX
- アドレス
- コマンド
- データ長
- データ部
- ETX

SUM計算に含めない:

- SUM自身
- CR

> 根拠: 第5章 5.2「通信フォーマットの詳細」本文p.99
> 根拠: 第5章 5.3「SUM の計算方法」本文p.100

## 計算方法

SUM値は、STXからETXまでのデータを1バイト単位で加算した結果の下位1バイト。

桁あふれが発生した場合は、あふれた桁を捨てた値をSUM値とする。

```text
SUM = sum(frame_bytes_from_stx_to_etx) & 0xFF
```

> 根拠: 第5章 5.3「SUM の計算方法」本文p.100

## 表現形式

- フレームはバイナリデータとして扱う。
- SUMも1バイトのバイナリ値として格納する。
- ASCII HEX文字列ではなく、実バイト列として送受信する。

> 根拠: 第5章 5.2「通信フォーマットの詳細」本文p.99

## 計算例1: リーダライタ動作モードの読み取り

SUM計算対象:

```text
02 00 4F 01 00 03
```

内訳:

| 項目 | 値 |
|---|---|
| STX | `02h` |
| アドレス | `00h` |
| コマンド | `4Fh` |
| データ長 | `01h` |
| データ部 | `00h` |
| ETX | `03h` |

計算結果:

```text
02h + 00h + 4Fh + 01h + 00h + 03h = 55h
```

SUMは `55h`。

> 根拠: 第5章 5.3「SUM の計算方法」本文p.100

## 計算例2: 汎用ポート値の書き込み

SUM計算対象:

```text
02 00 4E 03 9F 45 05 03
```

内訳:

| 項目 | 値 |
|---|---|
| STX | `02h` |
| アドレス | `00h` |
| コマンド | `4Eh` |
| データ長 | `03h` |
| データ部 | `9Fh 45h 05h` |
| ETX | `03h` |

計算結果:

```text
02h + 00h + 4Eh + 03h + 9Fh + 45h + 05h + 03h = 013Fh
```

下位1バイトを採用するため、SUMは `3Fh`。

> 根拠: 第5章 5.3「SUM の計算方法」本文p.100

## 検証方法

送信時:

1. STXからETXまでのバイト列を組み立てる。
2. SUMを計算する。
3. `STX...ETX + SUM + CR` の順に送信する。

受信時:

1. STX、ETX、CR、データ長を確認する。
2. 受信SUMを取り出す。
3. STXからETXまでを再計算する。
4. 受信SUMと再計算SUMが一致するか確認する。
5. 不一致の場合は受信フレームを異常として扱う。

> 根拠: 第5章 5.2「通信フォーマットの詳細」本文p.99
> 根拠: 第5章 5.3「SUM の計算方法」本文p.100
> 根拠: 第7章 7.6「NACK レスポンスとエラーコード」本文p.317

## 実装時の注意

- SUM計算対象にCRを含めない。
- SUM計算対象にSUM自身を含めない。
- STXとETXはSUM計算対象に含める。
- 桁あふれは捨て、下位1バイトのみ使用する。
- 受信フレームのSUM不一致は、通信異常として扱う。
- 上位機器からリーダライタへ不正なSUM値を送信した場合、リーダライタはNACKのエラーコード1として `42h: SUM_ERROR` を返す。

> 根拠: 第5章 5.3「SUM の計算方法」本文p.100
> 根拠: 第7章 7.6「NACK レスポンスとエラーコード」本文p.317
